# 포트폴리오 업데이트 가이드

이 문서는 코드를 수정한 후 GitHub에 변경 사항을 업로드하는 방법을 설명합니다.

## 1. 현재 상태 확인
어떤 파일이 수정되었는지 확인합니다.
```bash
git status
```

## 2. 변경 사항 저장 준비 (Stage)
수정한 모든 파일을 업로드 목록에 추가합니다.
```bash
git add .
```
특정 파일만 추가하고 싶다면:
```bash
git add 파일경로/파일이름
```

## 3. 커밋 (Save)
변경 사항에 이름을 붙여 저장합니다. 따옴표 안에 변경 내용을 간단히 적으세요.
```bash
git commit -m "수정 내용 (예: 프로젝트 정보 업데이트)"
```

## 4. 업로드 (Push)
저장한 내용을 GitHub 웹사이트로 보냅니다.
```bash
git push
```
또는 명확하게 지정하려면:
```bash
git push origin main
```

---

## ⚠️ 문제 해결: 업로드가 안 될 때

만약 `git push`가 실패한다면, 깃허브에 있는 내용이 로컬(내 컴퓨터)보다 최신 버전이라서 그럴 수 있습니다. 먼저 업데이트를 받아와야 합니다.

### 방법 1: 업데이트 받아오기 (Pull)
```bash
git pull origin main --rebase
```
이후 다시 `git push`를 시도하세요.

### 방법 2: 충돌이 발생했을 때
업데이트를 받아오는데 "CONFLICT (content)" 같은 메시지가 뜨면, 코드가 서로 달라서 컴퓨터가 자동으로 합치지 못한 것입니다.
이 경우 VS Code에서 충돌된 파일을 열어 올바른 내용을 선택한 저장한 뒤, 다음 과정을 진행하세요:

```bash
git add .
git commit -m "충돌 해결"
git push
```
